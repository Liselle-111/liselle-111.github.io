Getting Started

Pre-setup
  • You must install expo tools before setting up the project, assuming you have npm or yarn installed
  • npm install expo-cli --global or yarn global add expo-cli
Setup
  • git clone git@github.com:Liselle-111/Leg-Up.git
  • yarn to install dependencies
  • yarn start Starts the local server
  • ? to get more available options
Running: Device
  • Download expo app from the app store
  • Android: Scan the barcode that is printed after running yarn start
  • iOS: Login to your expo account on the expo app, the project will be listed after running yarn start
  • You must have an internet connection
Running: Simulator
  • yarn start
  • i for iOS or a Android simulators, this will install and run the app
  • yarn offline - this will allow you to run the simulator without internet
Running: Web
  • yarn web

Navigation

navigator.tsx

navigator.tsx contains returns an App Container which itself contains an Animated Switch Navigator, from here on referred to as the Root Navigator. This Root Navigator contains only sub-navigators. with each subnavigator relating to a specific area of the application.

As previously mentioned, each route in the Root Navigator is itself a navigator. Each of these sub-navigators contains screens related to a specific section of the application. This allows for navigation between features to be controlled by the Root Navigator, and navigation between screens of the same feature is controlled by this.props.navigation. More information about the navigation prop can be found here.

RootNavigation.ts

RootNavigation.ts returns an object with several functions relating to the Root Navigator component. In AppContainer.tsx the RefObject contained in RootNavigator.ts is assigned to the instance of navigator.tsx. This exposes the functions of the Root Navigator, and allows us to switch between the subnavigators it contains from anywhere in the application, simply by importing RootNavigator.ts.
